## 🎯 Phase 3: 数据库生产化任务

### 任务1: 数据库健康监控系统 ⭐⭐⭐⭐⭐

**优先级**: 🔥 最高
**预计时间**: 2-3小时
**交付物**:
- 实时健康检查仪表板
- 自动告警系统
- 性能指标追踪

#### 1.1 创建健康检查PHP类

**文件**: `inc/database/class-cyberpunk-db-health.php`

**功能需求**:
```php
class Cyberpunk_DB_Health {
    // 检查表状态
    public static function check_table_status()

    // 检查索引效率
    public static function check_index_usage()

    // 检查慢查询
    public static function check_slow_queries()

    // 检查连接数
    public static function check_connections()

    // 检查磁盘空间
    public static function check_disk_space()

    // 生成健康报告
    public static function generate_health_report()
}
```

**监控指标**:
- 表碎裂率 (应 < 10%)
- 索引使用率 (应 > 95%)
- 慢查询数量 (应 < 10/小时)
- 连接池使用率 (应 < 80%)
- 查询响应时间 (应 < 100ms)

#### 1.2 创建WordPress管理页面

**文件**: `inc/database/admin-db-health.php`

**功能**:
- 实时健康状态仪表板
- 图表展示性能趋势
- 一键优化按钮
- 告警历史记录

**位置**: WordPress后台 → 仪表板 → Cyberpunk DB Health

#### 1.3 邮件告警系统

**触发条件**:
- ❌ 表不可用
- ⚠️ 慢查询超过阈值
- ⚠️ 磁盘空间不足 20%

### 快速命令

```bash
# 查看当前数据库状态
mysql -u root -p -e "SHOW TABLES LIKE 'wp_cyberpunk%'"

# 检查慢查询
mysql -u root -p -e "SHOW VARIABLES LIKE 'slow_query%'"

# 查看表大小
mysql -u root -p -e "
SELECT
    TABLE_NAME,
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS size_mb
FROM information_schema.TABLES
WHERE table_schema = 'wordpress'
AND TABLE_NAME LIKE 'wp_cyberpunk%'
ORDER BY size_mb DESC;
"
```
